#  Портфолио по ручному тестированию

##  О себе

Начинающий тестировщик. В этом портфолио демонстрирую навыки:

- ✅ Составление тест-кейсов и чек-листов
- ✅ Поиск и оформление баг-репортов (Jira)
- ✅ Тест-дизайн (классы эквивалентности, граничные значения)

---

## Объект тестирования

**[SauceDemo](https://www.saucedemo.com/)** — демо-интернет-магазин, созданный для обучения тестированию.

Почему выбран этот сайт:
- Специально создан для тестирования
- Есть разные типы пользователей (standard_user, locked_out_user, problem_user, visual_user)
- Позволяет проверить полный цикл: авторизация → выбор товара → корзина → оформление

## Баг-репорты

Всего найдено: 5 дефектов

| ID | Заголовок | Серьезность | Приоритет |
|----|-----------|-------------|-----------|
| BUG-01-postal-code-validation | Поле Postal Code принимает невалидные значения (буквы, 11+ цифр) | Major | Medium |
| BUG-02-problem-user-same-images | Все товары имеют одинаковое изображение (problem_user) | Major | High |
| BUG-03-description-image-mismatch | Описание товара не соответствует изображению | Minor | Low |
| BUG-04-auto-logout | Автоматический выход из аккаунта через 10 минут бездействия | Major | High |
| BUG-05-visual-user-price-mismatch | Цены на главной странице не соответствуют ценам в карточке (visual_user) | Major | High |

**Статистика по серьезности:**
- Major: 4 бага
- Minor: 1 баг

**Статистика по приоритету:**
- High: 3 бага
- Medium: 1 баг
- Low: 1 баг

##  Как посмотреть портфолио

1. Тест-кейсы: откройте `test-cases/test-cases-saucedemo.csv` (можно просмотреть прямо на GitHub)
2. Баг-репорты: перейдите в папку `bug-reports/` и откройте любой `.md` файл, либо экспортируйте файл 'Jira' и откройте в Jira
3. Скриншоты: папка `bug-reports/screenshots/`

Чтобы воспроизвести баги:
1. Перейдите на SauceDemo (https://www.saucedemo.com/)
2. Используйте логины: `standard_user`, `problem_user`, `visual_user`
3. Пароль для всех: `secret_sauce`
4. Следуйте шагам из баг-репортов
